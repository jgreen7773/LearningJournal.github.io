I learned how to ignore files through the given command in the terminal
git rm --cached .DS_Store (to remove said file in current repository)
This next step is special because it makes a rule to ignore said file
echo .DS_Store >> ~/.gitignore_global
This last step is also special (and needed) because it uses -newly made rule- for ALL FILES
git config --global core.excludesfile ~/.gitignore_global

  1. What makes a computer, a computer?
    1. As people wanted to design a "thinking machine" it would need to:
    • Take input
    • Store information
    • Process it
    • Output the result

  1. How is information represented using electricity?
    1. First, electricity was coded as either a "1", or a "0"...depending on it's on-or-off, true-or-false, yes-or-no state.
    2. Second, a language had to be invented consisting only of "1's", or "0's".
    3. Last of all, the elements of a circuitboard accept and receive the language in sequences of numbers, kind of like words without letters.
    • The on-or-off, yes-or-no, true-or-false, 0-or-1 state represents "1 bit".
    • Any number can be represented using only 1's and 0's.
    • Text, images, and sound can also be represented using only 1's and 0's.

  1. How do computers use circuits?
    1. Different types of circuits have different functions:
    • Flipper circuit: changes 0(off) to 1(on) and 1(on) to 0(off), respectively.
    • And circuit: will only give you a 1(on) when both incoming signals are 1(on). If one is 0(off) it returns a 0(off).
    • Adder circuit: multiple "Adders" are put side by side to do math calculations, because 1 adder circuit can only calculate a maximum of two numbers.
    • There are many other types of circuits that are used for various other methods of translation.

  1. What are the different parts of the computer and how do they (input, output, store, and) process information?

  1. What is code? And how does software control hardware?
    • The encompassing answer to this question lies in the 4 step process of computing.
      1. The computer must have input via input devices such as keyboards, mice, remote controllers, cameras(self obtaining data conversion input--e.g. face recog software).
      2. Next, the computer receives the input commands through 1's and 0's (binary code) from things like tv remotes, keyboards, etc...and makes a "how-to" request to 'Memory'(storage).
      3. The request then goes to the CPU to interperet step-by-step "how-to" instructions on how to translate that "code".
      4. The resulting translation is what we lastly, call output. The "code" has been interpereted into a viable output operation or function.
